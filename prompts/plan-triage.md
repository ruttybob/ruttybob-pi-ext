# Plan Triage — ревизия планов и задач

Пройдись по всем планам и задачам (или указанной подгруппе), проверь статус каждого и перенеси реализованные в архив.

<goal>
$@
</goal>

## Что делать

1. Определи scope — что проверять:
   - `all` или пустой — планы + задачи
   - `plans` — только `.plans/` и `.plans/.detailed/`
   - `issues` — только `.issues/`
   - `<slug-or-pattern>` — планы и задачи, совпадающие по имени
2. Для каждого файла прочитай и определи статус
3. Перенеси реализованные в `_done`
4. Выведи сводку

## Как определить статус

Для каждого плана/задачи:

1. Прочитай файл целиком
2. Если есть строка `**Статус:** ✅ Реализовано` — реализовано
3. Если статус `⏳ Не начата` или `⏳ Не реализовано` — проверь кодовую базу:
   - Найди файлы, упомянутые в плане/задаче
   - Проверь: существуют ли они, содержит ли код описанные изменения
   - Проверь коммиты: `git log --oneline --grep="<ключевые-слова>"`
   - Проверь есть ли тесты и проходят ли они
4. Если явно реализовано — обнови статус на `✅ Реализовано`

**Приоритет проверки:**
- Статус в файле > анализ кодовой базы > git log
- Если сомневаешься — оставь на месте, отметь как "требует проверки"

## Перемещение файлов

Используй `mv`. Если целевая директория не существует — создай её.

**Планы:**
- `.plans/<file>.md` → `.plans/_done/<file>.md`
- `.plans/<slug>/<file>.md` → `.plans/_done/<slug>/<file>.md`
- `.plans/.detailed/<file>.md` → `.plans/.detailed/_done/<file>.md`

**Задачи:**
- `.issues/<slug>/<file>.md` → `.issues/_done/<file>.md`

## Сводка

Выведи две таблицы — планы и задачи:

```
## Планы

| План | Статус | Действие |
|------|--------|----------|
| 2026-05-03_mesh-overlay.md | ✅ Реализовано | → _done |
| 2026-05-04_evolver-refactor.md | ⏳ Не реализовано | оставлен |

## Задачи (.issues/remove-quota/)

| Задача | Статус | Действие |
|--------|--------|----------|
| 001-remove-tavily-quota.md | ✅ Реализовано | → _done |
| 002-remove-brave-quota.md | ⏳ Не начата | оставлена |
```

В конце — общая статистика:
- Сколько проверено (планов / задач)
- Сколько перенесено в _done
- Сколько осталось активных
- Сколько требует ручной проверки

## Правила

- Не удаляй файлы — только перемещай в _done
- Не модифицируй содержимое (кроме обновления поля статуса)
- Если частично реализовано — оставь на месте, укажи "частично"
- Папки `summary/`, `_done/` не трогай — они не являются планами/задачами
