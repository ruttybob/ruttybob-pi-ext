# Writing REPPI.md Templates

## Порядок секций (строго)

Каждую секцию оборачивать в парные XML-теги. Порядок фиксирован:

```markdown
<role>
Роль агента. Кто он, что делает, как общается.
Инструкции по поведению, стилю, ограничениям.
</role>

{{#if mode}}
<mode>
{{mode}}
</mode>
{{/if}}

<tools>
{{tools}}
</tools>

{{#if tool_guidelines}}
<guidelines>
{{tool_guidelines}}
</guidelines>
{{/if}}

{{#if documentation}}
<documentation>
{{documentation}}
</documentation>
{{/if}}

{{#if context_files}}
<context>
{{context_files}}
</context>
{{/if}}

{{#if skills}}
<skills>
{{skills}}
</skills>
{{/if}}
```

## Зачем XML-теги

Чёткие границы секций. Модель лучше понимает структуру и реже путает
инструкции из разных секций. Особенно важно при длинных промптах.

## Доступные переменные

| Переменная | Что подставляет |
|---|---|
| `{{tools}}` | Список активных инструментов с описаниями |
| `{{tool_guidelines}}` | Инструкции по использованию инструментов |
| `{{mode}}` | Текущий mode (имя + инструкции пресета) |
| `{{documentation}}` | Ссылки на документацию pi (обычно пустой при REPPI) |
| `{{skills}}` | Загруженные скиллы |
| `{{context_files}}` | AGENTS.md / CLAUDE.md |
| `{{base_prompt}}` | Полный дефолтный промпт pi |
| `{{model_info}}` | Имя модели и провайдера |
| `{{date}}` | Текущая дата |
| `{{cwd}}` | Рабочая директория |

## Условные блоки

`{{#if var}}...{{/if}}` — секция появляется только если переменная непустая.
Без тегов — пустая строка + заголовок остаются в промпте, тратя токены.

## Чего избегать

- Не использовать `{{base_prompt}}` внутри кастомного шаблона — дублирование
- Не добавлять date и cwd вручную — pi добавляет их автоматически в конце
- Не использовать `{{documentation}}` — при REPPI дефолтный промпт не генерируется, переменная пустая. Если нужны ссылки — добавь статически
